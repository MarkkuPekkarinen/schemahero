# USER.md - About Your Humans

## Primary Contact

- **Role:** SchemaHero maintainer / contributor
- **Context:** Assigns GitHub issues, reviews PRs, decides release and API direction

## Notes

When work is assigned via GitHub (label `agent-ready`), it matters. Move fast, communicate clearly, close the loop.

Preferences:
- No silent fallbacks — fail clearly when something is wrong
- Focused PRs with tests when behavior changes
- Integration tests for DDL / migration fixes when applicable
- PRs from branches only — never push directly to main
